Interpretado? Compilado?

R é uma linguagem interpretada no sentido em que não é compilada… Linguagens compiladas são traduzidas numa outra.

Como por exemplo C! 

#include <stdio.h>
void main() {
  printf("olá olá!\n");
}

olá olá!

Ou Fortran! 

program ola
  print *, 'olá, olá!'
end program ola

olá, olá!

Um binário!

Muitas vezes compilado significa de forma coloquial "traduzido para zeros e uns". C e Fortran são linguagens que compilamos para binários! 

gcc ./src/ola.c -o ./src/ola.exe

No computador só há zeros e uns?

Efetivamente as máquinas que usamos tendem a preferir digerir informação em formato binário. Por exemplo as imagens em formato png! 

xxd -b grasshopper.png | head -n1 | cut -d' ' -f2-

Mas e o R?

O R como disse é interpretada, em vez de ser traduzido para binário, as instruções do R são lidas e executadas diretamente pelo seu interpretador! 

xxd -b /usr/lib/R/bin/Rscript | head -n1 | cut -d' ' -f2-

01111111 01000101 01001100 01000110 00000010 00000001  .ELF..

Efetivamente podemos validar que o interpretador do R é um interpretador… 

file /usr/lib/R/bin/Rscript | cut -d' ' -f 2,3,6,13

ELF 64-bit executable, interpreter

Múltiplos paradigmas?

Existem muitos, e o que cada um significa, ou não, é um assunto bastante debatido…

Podemos no entanto falar de dois importantes:

  • Imperativo
  • Declarativo

Imperativo?

Imperativo no sentido em que temos instruções que alteram o estado do programa. Uma espécie de receita que o computador segue um por um. 

numbers <- c(1, 2, 3, 4, 5)
squared_numbers <- numeric(length(numbers))

for (i in 1:length(numbers)) {
  squared_numbers[i] <- numbers[i]^2
}

print(squared_numbers)

[1]  1  4  9 16 25

Declarativo?

No sentido em que dizemos ao computador o que gostávamos de observar como resultado final mas sem nos importarmos com o "como chegar ao resultado final". 

print(c(1:5)^2)

[1]  1  4  9 16 25

Imperativo no Fortran por exemplo

Fortran também tem imperativo… 

program imperative_example
  implicit none
  integer, parameter :: n = 5
  integer :: i
  integer :: numbers(n) = (/1, 2, 3, 4, 5/)
  integer :: squared_numbers(n)

  do i = 1, n
    squared_numbers(i) = numbers(i) ** 2
  end do

  write(*, '(5I3)') squared_numbers

end program imperative_example

1  4  9 16 25

Declarativo no Fortran por exemplo

E declarativo… 

program declarative_example
  implicit none
  integer :: i
  integer, parameter :: numbers(5) = (/ (i ** 2, i=1,5) /)
  write(*, '(5I3)') numbers
end program declarative_example

Tipos de coisas

No R temos seis tipos de dados "básicos":

O quê? Como assim de outras linguagens?

Sim podemos usar funções de fortran em R por exemplo! 

subroutine quadrado(x, result)
  implicit none
  integer, intent(in) :: x
  integer, intent(out) :: result

  result = x * x
end subroutine quadrado

Mas como?

Compilando o programa de fortran de uma forma especial… 

gfortran -shared -fPIC -o src/aoquadrado.so src/aoquadrado.f90

E importando no R! 

dyn.load("src/aoquadrado.so")
result <- .Fortran("quadrado", as.integer(3), integer(1))
print(result[[2]])

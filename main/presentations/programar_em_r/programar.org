** Imperativo?
Imperativo no sentido em que temos instruções que alteram o estado do programa.
Uma espécie de _receita que o computador segue um por um_.

#+begin_src R :results output :exports both
  numbers <- c(1, 2, 3, 4, 5)
  squared_numbers <- numeric(length(numbers))

  for (i in 1:length(numbers)) {
    squared_numbers[i] <- numbers[i]^2
  }

  print(squared_numbers)
#+end_src

#+RESULTS:
: [1]  1  4  9 16 25

** Declarativo?
No sentido em que dizemos ao computador o que gostávamos de observar
como resultado final mas _sem nos importarmos com o "como chegar ao
resultado final"_.

#+begin_src R :results output :exports both
  print(c(1:5)^2)
#+end_src

#+RESULTS:
: [1]  1  4  9 16 25

#+begin_src R :results output :exports both :session
  library(compiler)

  # Uma função sem grande utilidade aparente
  # Mas que poderia aparecer de várias outras

  slow_fun <- function(n) {
    x <- 0
    for (i in 1:n) {
      x <- x + sqrt(i)
    }
    x
  }

** O que se passa aqui?
#+REVEAL_HTML:<div style="text-align: left;">
O Interpretador de R está a traduzir o que escrevemos para uma
linguagem mais "baixo nível". Esta linguagem não é tão cómoda para nós
lermos/escrevermos mas ao traduzir antes de usarmos, fazemos com que o
interpretador de R "tenha menos trabalho".
#+REVEAL_HTML:</div>

  f_compiled <- cmpfun(f)

  disassemble(f_compiled)
